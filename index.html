Privacy-Preserving Data Publishing

t-Closeness

Complete Numerical Question Bank — All possible types of computational problems with detailed step-by-step solutions

Core Concepts Review

t-Closeness requires that the distribution of a sensitive attribute in any equivalence class is within distance t of the distribution of the attribute in the overall table. It uses the Earth Mover's Distance (EMD) to measure this.

EMD (for categorical) = ½ × Σ |pi − qi|
EMD (for numerical / ordered) = (1 / (m−1)) × Σi=1m | Σj=1i (rj − sj) |

Where:

  • pi, qi — fraction of value i in the equivalence class vs. overall table
  • rj, sj — fraction of the j-th ordered value in class vs. overall table
  • m — number of distinct values of the sensitive attribute

2 EMD for Numerical (Ordered) Sensitive Attributes

Compute EMD using the ordered / numerical distance formula involving cumulative differences. Used when the sensitive attribute is Salary, Age, etc.

Q4 A table has Salary as the sensitive attribute with ordered values {10K, 20K, 30K, 40K}. Overall distribution: each value appears 3 times (total 12). An equivalence class has 4 records: {10K, 10K, 30K, 40K}. Compute the numerical EMD. Medium
Solution
Step 1: Overall distribution (s): s(10K) = 3/12 = 1/4, s(20K) = 1/4, s(30K) = 1/4, s(40K) = 1/4
Step 2: Class distribution (r): r(10K) = 2/4 = 1/2, r(20K) = 0, r(30K) = 1/4, r(40K) = 1/4
Step 3: Differences (r − s):
d₁ = 1/2 − 1/4 = 1/4
d₂ = 0 − 1/4 = −1/4
d₃ = 1/4 − 1/4 = 0
d₄ = 1/4 − 1/4 = 0
Step 4: Cumulative sums:
C₁ = 1/4
C₂ = 1/4 + (−1/4) = 0
C₃ = 0 + 0 = 0
C₄ = 0 + 0 = 0
Step 5: Absolute cumulative sums: |1/4|, |0|, |0|, |0| → Sum = 1/4
Step 6: EMD = 1/(m−1) × Sum = 1/(4−1) × 1/4 = 1/12 = 0.0833
Q5 Overall salary distribution: {3K: 3, 5K: 2, 7K: 4, 9K: 1} (total = 10). Equivalence class: {3K, 5K, 7K, 9K} (4 records). Compute the numerical EMD. Medium
Solution
Step 1: Overall (s): s(3K)=3/10, s(5K)=2/10, s(7K)=4/10, s(9K)=1/10
Step 2: Class (r): r(3K)=1/4, r(5K)=1/4, r(7K)=1/4, r(9K)=1/4
Step 3: Differences (r − s):
d₁ = 1/4 − 3/10 = 5/20 − 6/20 = −1/20
d₂ = 1/4 − 2/10 = 5/20 − 4/20 = 1/20
d₃ = 1/4 − 4/10 = 5/20 − 8/20 = −3/20
d₄ = 1/4 − 1/10 = 5/20 − 2/20 = 3/20
Step 4: Cumulative sums:
C₁ = −1/20
C₂ = −1/20 + 1/20 = 0
C₃ = 0 + (−3/20) = −3/20
C₄ = −3/20 + 3/20 = 0
Step 5: Sum of |C| = 1/20 + 0 + 3/20 + 0 = 4/20 = 1/5
Step 6: EMD = 1/(4−1) × 1/5 = 1/15 = 0.0667

8 EMD with Binary Sensitive Attributes

Special case where the sensitive attribute has only two values (e.g., Yes/No, Positive/Negative).

Q13 A medical study has 100 records where "Test Result" is the sensitive attribute: Positive = 30, Negative = 70. An equivalence class of 10 records has 6 Positive and 4 Negative. Compute the EMD. Easy
Solution
Step 1: Overall: P(+) = 30/100 = 0.30, P(−) = 70/100 = 0.70
Step 2: Class: P(+) = 6/10 = 0.60, P(−) = 4/10 = 0.40
Step 3: EMD = ½ × (|0.60 − 0.30| + |0.40 − 0.70|) = ½ × (0.30 + 0.30) = 0.30
For binary attributes, EMD always simplifies to: EMD = |p₁ − q₁| (the absolute difference for one value) since both differences are always equal.

9 Theoretical Maximum EMD

Compute the worst-case EMD — what is the maximum possible EMD for a given overall distribution?

Q14 If the overall distribution of a sensitive attribute with 4 values is {0.25, 0.25, 0.25, 0.25}, what is the maximum possible EMD for any equivalence class? Medium
Solution
Step 1: The worst case is when the class has all records with a single value. E.g., class distribution = {1, 0, 0, 0}.
Step 2: EMD = ½ × (|1−0.25| + |0−0.25| + |0−0.25| + |0−0.25|)
= ½ × (0.75 + 0.25 + 0.25 + 0.25) = ½ × 1.50 = 0.75
For a uniform distribution over m values, the maximum categorical EMD = (m−1)/m. Here: (4−1)/4 = 3/4 = 0.75.

Solution

Case 2: Class = all Disease2 → {0, 1.0}

11 Effect of Adding/Removing Records

Determine how adding or removing a record from an equivalence class changes the EMD.

Q17 An equivalence class has {Flu, Cancer, HIV} (3 records). Overall: P(Flu)=P(Cancer)=P(HIV)=1/3. If we add one more "Flu" record to this class, how does the EMD change? (Assume overall distribution stays the same.) Medium

Result: EMD increased from 0 → 0.167. Adding a duplicate sensitive value significantly increases distance.

12 t-Closeness vs. l-Diversity Failure

Show scenarios where a table satisfies l-diversity but fails t-closeness (or vice versa).

Q18 A table has Overall: {Flu:1, Cancer:1, HIV:8} (10 records). An equivalence class has: {Flu, Cancer, HIV}. Show that it satisfies 3-diversity but compute the EMD. Medium
Solution
l-Diversity: The class has 3 distinct values → ✓ Satisfies 3-diversity
t-Closeness:
Overall: P(Flu)=0.10, P(Cancer)=0.10, P(HIV)=0.80
Class: P(Flu)=1/3≈0.333, P(Cancer)=1/3≈0.333, P(HIV)=1/3≈0.333
EMD = ½ × (|0.333−0.10| + |0.333−0.10| + |0.333−0.80|)
= ½ × (0.233 + 0.233 + 0.467) = ½ × 0.933 = 0.467
Result: The class is 3-diverse but has EMD = 0.467, which is very high. It would fail t-closeness for any t < 0.467.
Key insight: l-diversity only counts distinct values. t-closeness checks how the distribution matches the overall table. A class can be l-diverse but completely skewed relative to the global distribution.

14 Computing Information Gain with t-Closeness

Compute the information gain (KL divergence or similar) an attacker gets, and relate it to the EMD threshold.

Q20 Overall: P(Flu)=0.5, P(HIV)=0.5. A class has: P(Flu)=0.8, P(HIV)=0.2. Compute the EMD and the KL-divergence to quantify the attacker's information gain. Hard
Solution
EMD:
EMD = ½ × (|0.8−0.5| + |0.2−0.5|) = ½ × (0.3+0.3) = 0.30
KL-Divergence: D_KL(Class ‖ Overall) = Σ p(x) × log₂(p(x)/q(x))
= 0.8 × log₂(0.8/0.5) + 0.2 × log₂(0.2/0.5)
= 0.8 × log₂(1.6) + 0.2 × log₂(0.4)
= 0.8 × 0.678 + 0.2 × (−1.322)
= 0.543 − 0.264 = 0.278 bits
Interpretation: By knowing someone is in this class, an attacker gains 0.278 bits of information about their disease. The EMD of 0.30 gives a bound on this gain.

d₅ = 0.20 − 0.15 = 0.05
Step 4: Cumulative sums:
C₁ = 0.30
C₂ = 0.30 + 0.00 = 0.30
C₃ = 0.30 + (−0.30) = 0.00
C₄ = 0.00 + (−0.05) = −0.05
C₅ = −0.05 + 0.05 = 0.00
Step 5: Sum of |C| = 0.30 + 0.30 + 0.00 + 0.05 + 0.00 = 0.65
Step 6: EMD = 1/(5−1) × 0.65 = 0.65/4 = 0.1625

17 Minimum Class Size to Guarantee t-Closeness

How large must an equivalence class be to guarantee a certain t value under worst-case distribution?

Q23 The overall distribution is uniform across 3 values: P(A) = P(B) = P(C) = 1/3. If we want t ≤ 1/3, what is the minimum class size (assuming worst case)? Medium
Solution
Analysis: The worst case for a class of size n is when one value dominates maximally. For a class of size n with 3 possible values, the worst case is one value appearing ⌈n×1⌉ = as many times as possible.
Key insight: If the class has exactly one of each value (size 3), the distribution matches the overall perfectly → EMD = 0. If size = 2, worst case is {A, A}: P(A)=1, P(B)=P(C)=0, EMD = ½×(|1−1/3|+|0−1/3|+|0−1/3|) = ½×(2/3+1/3+1/3) = 2/3.
Size = 3: Worst case {A, A, B}: P(A)=2/3, P(B)=1/3, P(C)=0. EMD = ½×(|2/3−1/3|+|1/3−1/3|+|0−1/3|) = ½×(1/3+0+1/3) = 1/3. This equals t.
Result: Minimum class size = 3 (and even then, worst case exactly hits t = 1/3). For strict guarantee, we need size ≥ 3.

18 Recursive (c,t)-Closeness

A variant where each equivalence class must be further partitioned and sub-groups must also satisfy t-closeness.

Q24 An equivalence class of 6 records has Disease: {Flu, Flu, Cancer, Cancer, HIV, HIV}. This is split into two sub-groups of 3 each: S1={Flu, Flu, Cancer}, S2={Cancer, HIV, HIV}. Overall: P(Flu)=P(Cancer)=P(HIV)=1/3. Compute EMD for each sub-group against the overall. Medium
Solution
Full class EMD: P(Flu)=2/6=1/3, P(Cancer)=2/6=1/3, P(HIV)=2/6=1/3
EMD = 0 (matches overall perfectly)
S1: P(Flu)=2/3, P(Cancer)=1/3, P(HIV)=0
EMD = ½ × (|2/3−1/3| + |1/3−1/3| + |0−1/3|) = ½ × (1/3+0+1/3) = 1/3 ≈ 0.333
S2: P(Flu)=0, P(Cancer)=1/3, P(HIV)=2/3
EMD = ½ × (|0−1/3| + |1/3−1/3| + |2/3−1/3|) = 1/3 ≈ 0.333
Result: While the full class has EMD = 0, its sub-groups have EMD = 1/3. This is the recursive closeness concern — sub-groups may reveal more than the parent class.

19 Skewness & Similarity Attack Detection

Demonstrate the attacks that t-closeness defends against using numerical calculations.

Q25 Overall: {Flu: 5, Cancer: 4, HIV: 1} (10 total). Class Y has 3 records: {Cancer, Cancer, Cancer}. Show the skewness attack numerically and how t-closeness detects it. Medium
Solution
l-Diversity check: Class Y has only 1 distinct value → Fails even 2-diversity. But let's assume a different class that IS l-diverse but still has skewness issues...
Modified Class Y': {Cancer, Cancer, HIV} (2-diverse)
Overall: P(Flu)=0.50, P(Cancer)=0.40, P(HIV)=0.10
Class: P(Flu)=0, P(Cancer)=2/3≈0.667, P(HIV)=1/3≈0.333
Skewness Attack: An attacker knows someone is in this class. Even though l-diversity holds (2 distinct values), the attacker knows there's a 0% chance of Flu — this eliminates a disease that has 50% probability overall. They also know there's a 66.7% chance of Cancer vs. 40% overall.
EMD detects this: EMD = ½ × (|0−0.50| + |0.667−0.40| + |0.333−0.10|)
= ½ × (0.50 + 0.267 + 0.233) = ½ × 1.00 = 0.50
Result: EMD = 0.50 is very high — t-closeness correctly flags this as a privacy risk even though l-diversity was satisfied.

20 Computing t for Merged Equivalence Classes

When two classes are merged, compute the new EMD and compare with the individual class EMDs.

Q26 Class C1 has 3 records: {Flu, Flu, Cancer}. Class C2 has 3 records: {Cancer, HIV, HIV}. Overall: P(Flu)=P(Cancer)=P(HIV)=1/3. Compute EMD before and after merging C1 and C2. Medium
Solution
EMD(C1): P(Flu)=2/3, P(Cancer)=1/3, P(HIV)=0
= ½ × (1/3 + 0 + 1/3) = 1/3 ≈ 0.333
EMD(C2): P(Flu)=0, P(Cancer)=1/3, P(HIV)=2/3
= ½ × (1/3 + 0 + 1/3) = 1/3 ≈ 0.333
Merged (C1∪C2): 6 records: {Flu:2, Cancer:2, HIV:2}
P(Flu)=2/6=1/3, P(Cancer)=1/3, P(HIV)=1/3
EMD = ½ × (0 + 0 + 0) = 0
Result: Merging reduced EMD from 0.333 → 0. Merging complementary classes improves t-closeness but reduces utility (more generalization).
